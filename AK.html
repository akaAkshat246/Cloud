The Platform

Cloud Storage, Rebuilt From First Principles

CloudVault is a full-stack, end-to-end encrypted personal cloud storage platform — spanning Android, Web, and a production-grade backend infrastructure.

Think Google Drive. Think OneDrive. Now strip out the surveillance, rebuild the UX from scratch, and price it for the people who actually use it. That is CloudVault — not a reskin, not a clone, but a ground-up platform built on the conviction that privacy should be the default, not a premium add-on.

"Every file is encrypted on your device before it ever touches our servers. We never see your data. Nobody does."

Three Platforms. One Product.

📱

Android

Built with Kotlin and Jetpack Compose. Background sync, Room DB for local state, Retrofit for API calls, and on-device AES-256-GCM encryption before any file leaves your phone.

🌐

Web App

React 18 and TypeScript. Drag-and-drop uploads, client-side encryption via libsodium.js, responsive design, and a dashboard built for clarity and speed.

⚙️

Backend

Node.js and Express REST APIs. JWT authentication, PostgreSQL with TypeORM, AWS S3 for file storage, and GitHub Actions keeping everything in sync.

Security

We Never See Your Data. Nobody Does.

End-to-end encryption is not a feature we added — it is the founding principle around which every architectural decision has been made.

Most cloud storage platforms encrypt your files in transit and at rest — but they hold the keys. Which means they can read your data, hand it to advertisers, or comply with a government request. CloudVault is built on a different premise: your encryption key never leaves your device. Period.

"Encryption is not a layer. It is the foundation."

How It Works

🔐

AES-256-GCM Encryption

Every file is encrypted locally using AES-256-GCM — the gold standard in symmetric encryption, used by governments and financial institutions worldwide. The encrypted ciphertext is what gets uploaded. The plaintext never leaves your device.

🧬

PBKDF2 Key Derivation

Your encryption key is derived from your password using PBKDF2 — a password-based key derivation function that makes brute-force attacks computationally infeasible. Even if an attacker had your encrypted file, cracking the key would take centuries.

🌐

libsodium.js on the Web

The web app uses libsodium.js — a battle-tested cryptographic library — to perform all encryption operations directly in your browser. No server-side decryption. No middlemen. The encrypted blob is all we ever receive.

🪙

JWT + bcrypt Authentication

Authentication is handled with JSON Web Tokens and bcrypt password hashing. Passwords are never stored in plaintext. Tokens are short-lived and scoped. Your identity and your data are protected independently.

🔍

Security Review Built In

CloudVault has a dedicated Security Reviewer role on the team — responsible for auditing the encryption implementation, reviewing API security, and conducting penetration testing before launch. Security is not a checkbox. It is a team function.

Our Promise

CloudVault will never sell your data. CloudVault will never read your files. CloudVault will never provide server-side decryption — because architecturally, we cannot. The keys live with you. The data is yours.
